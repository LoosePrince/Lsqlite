# 管理后台使用说明

## 登录

使用环境变量中的 `ADMIN_USER` 和 `ADMIN_PASSWORD` 登录后台。

开发环境默认地址：`http://localhost:5173`。

生产构建后默认由 Node.js 服务提供后台页面。

## 创建数据库

创建数据库时需要填写：

- 数据库名称。
- 可选 key。如果留空，系统生成随机 key。
- 可选备注。

创建完成后会显示数据库 key。服务端只保存 key 摘要，不保存明文 key，请立即保存。

## 数据库列表

左侧列表支持按状态筛选：

- `active`：可被外部 key 访问。
- `disabled`：保留元数据和文件，但外部 key 不可访问。
- `deleted`：软删除状态，默认列表不显示，文件保留，可恢复。
- `all`：显示全部状态。

详情页会显示文件路径、文件大小、最后访问时间和备注。

## 基础信息管理

详情页“基础信息”支持：

- 修改名称。
- 修改备注。
- 在 `active` 和 `disabled` 之间切换。
- 轮换 key。

轮换 key 后旧 key 立即失效。调用方需要更新 `Authorization: Bearer <database_key>`。

## 删除、恢复、永久删除

删除策略是 soft-first：

1. 点击“删除”后进入 `deleted` 状态。
2. 软删除数据库不会删除 SQLite 文件。
3. 软删除后外部 key 立即不可用。
4. 在 `deleted` 筛选中可以恢复。
5. “永久删除”会删除元数据和 SQLite 文件，需要输入数据库名称确认。

永久删除不可恢复，应先确认备份和影响范围。

## 表结构可视化管理

详情页“表结构”支持：

- 查看表、视图、字段、索引和行数。
- 创建表。
- 删除表。
- 添加字段。
- 创建索引。
- 删除索引。

建表字段使用 JSON 描述：

```json
[
  { "name": "id", "type": "integer", "primaryKey": true },
  { "name": "name", "type": "text", "notNull": true },
  { "name": "created_at", "type": "text", "defaultValue": "CURRENT_TIMESTAMP" }
]
```

添加字段示例：

```json
{ "name": "note", "type": "text" }
```

创建索引示例：

```json
{ "name": "idx_items_name", "columns": ["name"] }
```

SQLite 稳定支持的改表范围有限。复杂操作，例如重命名字段、删除字段、重建表结构，建议在 SQL 控制台中手动执行，并先备份数据库。

## 表数据可视化管理

详情页“数据浏览”支持：

- 按表分页查看数据。
- 插入一行。
- 按条件更新行。
- 按条件删除行。

插入和更新值使用 JSON：

```json
{ "name": "demo", "note": "hello" }
```

更新和删除条件使用 JSON：

```json
{ "id": 1 }
```

更新和删除必须提供条件，服务端会拒绝空条件。

## SQL 控制台

SQL 控制台用于高级维护，并提供常用示例：

- schema 查询。
- 分页查询。
- 插入。
- 更新。
- 删除。

管理员 SQL 控制台权限高于外部 key API。执行破坏性 SQL 前应确认目标数据库、条件和影响行数。

## 接口示例

详情页“接口示例”会展示当前数据库对应的管理接口路径，以及外部 key API 的 curl 示例。

## 审计日志

后台记录关键管理行为：

- 登录。
- 创建数据库。
- 修改数据库。
- 删除、恢复、永久删除。
- 轮换 key。
- 表结构变更。
- 表数据变更。
- 管理员 SQL 执行。

审计日志用于回溯管理操作，不保存数据库 key 明文。
